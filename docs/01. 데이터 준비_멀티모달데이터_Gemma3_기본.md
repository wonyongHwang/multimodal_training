이 코드는 **필라테스 이미지 분류 데이터와 Alpaca 형식의 텍스트 데이터를 동일한 스키마로 통합한 뒤 Hugging Face Hub에 업로드**하는 프로그램입니다.

앞서 살펴본 OCR 프로그램과 목적은 비슷하지만, 이미지와 라벨을 연결하는 방식이 다릅니다.

| 구분        | OCR 프로그램                | 현재 필라테스 프로그램        |
| --------- | ----------------------- | ------------------- |
| 이미지 저장 방식 | 여러 ZIP 파일 내부            | 일반 이미지 파일           |
| 라벨 저장 방식  | 이미지별 JSON               | 하나의 `labels.csv`    |
| 이미지 연결 키  | JSON의 `identifier`와 파일명 | CSV의 `image_path`   |
| 정답 생성     | `bbox[].data` 결합        | `pose_label`로 문장 생성 |
| 대용량 처리    | Arrow shard 단위          | 전체를 한 번에 메모리에 적재    |
| 중단 후 재개   | 가능                      | 지원하지 않음             |

# 1. 전체 목적

프로그램은 다음 두 데이터를 통합합니다.

```text
필라테스 이미지 + 동작 라벨
                ↓
      이미지 분류 학습 데이터

Alpaca 텍스트 데이터
                ↓
      일반 텍스트 지시 학습 데이터
```

최종적으로 하나의 Hugging Face Dataset 안에 다음 두 유형의 행이 함께 들어갑니다.

```text
image_text
- 이미지 있음
- 이미지의 필라테스 동작명을 출력

text_only
- 이미지 없음
- instruction과 input에 대한 텍스트 응답 출력
```

전체 실행 흐름은 다음과 같습니다.

```text
.env 설정 로드
    ↓
Alpaca 텍스트 JSON 로드
    ↓
labels.csv 로드
    ↓
CSV의 image_path로 이미지 파일 읽기
    ↓
pose_label을 이용해 정답 문장 생성
    ↓
이미지 Dataset 생성
    ↓
텍스트 Dataset의 스키마를 이미지 Dataset에 맞춤
    ↓
이미지와 텍스트 Dataset 병합
    ↓
전체 셔플
    ↓
Hugging Face Hub 업로드
```

# 3. `labels.csv`의 예상 구조

코드에서 사용하는 CSV 컬럼은 두 개입니다.

```text
image_path
pose_label
```

예시는 다음과 같습니다.

```csv
image_path,pose_label
hundred/hundred_001.jpg,Hundred
hundred/hundred_002.jpg,Hundred
roll_up/roll_up_001.jpg,Roll Up
roll_up/roll_up_002.jpg,Roll Up
teaser/teaser_001.jpg,Teaser
```

각 컬럼의 의미는 다음과 같습니다.

| 컬럼           | 의미                             |
| ------------ | ------------------------------ |
| `image_path` | `IMAGE_ROOT`를 기준으로 한 이미지 상대 경로 |
| `pose_label` | 해당 이미지의 필라테스 동작명               |

코드는 다음과 같이 전체 이미지 경로를 만듭니다.

```python
imagePath = os.path.join(
    cfg["image_root"],
    df.loc[i, "image_path"]
)
```

예를 들어 설정이 다음과 같고,

```text
IMAGE_ROOT=./dataset/04. 멀티모달 샘플/image_samples
```

CSV 값이 다음과 같다면,

```text
hundred/hundred_001.jpg
```

최종 경로는 다음과 같습니다.

```text
./dataset/04. 멀티모달 샘플/image_samples/hundred/hundred_001.jpg
```

즉, 이 프로그램에서는 CSV 자체가 이미지와 라벨을 연결하는 역할을 합니다.

```text
CSV 한 행
├── 이미지 파일 위치
└── 해당 이미지의 정답 동작명
```

OCR 코드에서 JSON의 `identifier`를 사용해 ZIP 내부 이미지를 찾았던 것보다 훨씬 단순한 구조입니다.

# 4. 텍스트 JSON의 예상 구조

텍스트 데이터는 Alpaca 형식의 JSON Lines 파일입니다.

```json
{"instruction":"다음 질문에 답하세요.","input":"필라테스란 무엇인가요?","output":"필라테스는 신체의 균형과 코어 근육 강화를 목적으로 하는 운동입니다."}
{"instruction":"문장을 요약하세요.","input":"필라테스는 호흡과 자세를 중요하게 생각합니다.","output":"필라테스는 호흡과 자세를 중시합니다."}
```

`lines=True`를 사용하므로 전체 파일이 하나의 JSON 배열인 형식이 아니라, **한 줄에 하나의 JSON 객체**가 있어야 합니다.

```python
pd.read_json(
    cfg["text_json_path"],
    lines=True
)
```

올바른 형식:

```json
{"instruction":"...","input":"...","output":"..."}
{"instruction":"...","input":"...","output":"..."}
```

현재 코드와 맞지 않는 형식:

```json
[
  {"instruction":"...","input":"...","output":"..."},
  {"instruction":"...","input":"...","output":"..."}
]
```

# 9. 클래스별 샘플 수 제한

```python
if cfg["max_image_samples_per_class"] is not None:
```

클래스별 최대 샘플 수가 설정된 경우 현재 개수를 확인합니다.

```python
currentCount = classCounts.get(
    label,
    0
)
```

아직 등장하지 않은 클래스는 0으로 시작합니다.

최대 개수에 도달했다면 건너뜁니다.

```python
if currentCount >= cfg["max_image_samples_per_class"]:
    continue
```

아직 제한에 도달하지 않았다면 카운트를 증가시킵니다.

```python
classCounts[label] = currentCount + 1
```

예를 들어 제한이 2이고 CSV가 다음과 같다면,

```text
Hundred 이미지 1
Hundred 이미지 2
Hundred 이미지 3
Roll Up 이미지 1
Roll Up 이미지 2
Roll Up 이미지 3
```

선택 결과는 다음과 같습니다.

```text
Hundred 이미지 1
Hundred 이미지 2
Roll Up 이미지 1
Roll Up 이미지 2
```

각 클래스에서 앞쪽 2개만 선택됩니다.

## 중요한 특징

이 방식은 무작위 선택이 아닙니다.

```text
CSV에 먼저 등장한 N개 샘플
```

만 선택합니다.

따라서 CSV가 촬영 날짜, 사람, 카메라 각도 순서로 정렬되어 있으면 데이터 편향이 발생할 수 있습니다.

`random.seed()`를 설정하고 있지만 현재 이미지 샘플 선택에는 `random`이 사용되지 않습니다.

# 11. 이미지 정답 생성

```python
outputText = (
    f"동작명: {label}\n"
    f"설명: 이 이미지는 {label} 동작 예시입니다."
)
```

예를 들어 라벨이 `Hundred`라면 다음 정답이 만들어집니다.

```text
동작명: Hundred
설명: 이 이미지는 Hundred 동작 예시입니다.
```

즉, 실제 설명 데이터가 별도로 있는 것이 아니라 동작 라벨을 이용해 고정 템플릿으로 문장을 생성합니다.

```text
동작명: [pose_label]
설명: 이 이미지는 [pose_label] 동작 예시입니다.
```

따라서 이 데이터로 모델이 학습하는 핵심은 사실상 **이미지 동작 분류**입니다.

설명 문장은 모든 클래스에서 동일한 형식이기 때문에 모델이 필라테스 동작의 자세나 특징을 자세히 설명하도록 학습되는 것은 아닙니다.

예를 들어 다음처럼 각 동작에 대한 설명 메타데이터가 있어야 보다 의미 있는 설명 생성 학습이 가능합니다.

```csv
image_path,pose_label,description
hundred/001.jpg,Hundred,누운 상태에서 다리를 들어 올리고 팔을 위아래로 반복하는 동작입니다.
```

---

## 14.1 텍스트 Dataset의 스키마 변환

```python
textDataset = textDataset.cast(
    imageDataset.features
)
```

이 부분은 매우 중요합니다.

텍스트 Dataset의 `image` 컬럼에는 모두 `None`만 들어 있습니다. 따라서 Dataset 생성 과정에서 해당 컬럼이 이미지 타입이 아니라 `null` 타입으로 추론될 수 있습니다.

반면 이미지 Dataset에서는 `image` 컬럼이 Hugging Face `Image` 타입입니다.

개념적으로 다음처럼 타입 차이가 발생할 수 있습니다.

```text
imageDataset
image: Image

textDataset
image: null
```

문자열 컬럼도 Dataset 버전에 따라 다음처럼 다를 수 있습니다.

```text
imageDataset.output: string
textDataset.output: large_string
```

`concatenate_datasets()`는 컬럼 이름뿐만 아니라 컬럼 타입도 호환되어야 합니다.

따라서 다음 코드로 텍스트 Dataset의 feature를 이미지 Dataset에 강제로 맞춥니다.

```python
textDataset.cast(
    imageDataset.features
)
```

변환 후에는 텍스트 데이터의 `image=None`도 `Image` 타입 컬럼 안의 결측값으로 처리됩니다.

---

## 14.2 Dataset 병합

```python
allDataset = concatenate_datasets([
    imageDataset,
    textDataset
])
```

Dataset을 행 방향으로 연결합니다.

예를 들어 다음과 같습니다.

```text

textDataset:    500건
```

병합 결과:

```text

```

초기 순서는 다음과 같습니다.

```text
0~999       이미지 데이터
1000~1499   텍스트 데이터
```

# 17. OCR 프로그램과의 핵심 차이

## OCR 프로그램

```text
라벨 JSON
├── 이미지 identifier
├── bbox 텍스트
└── media_type

ZIP 이미지
└── identifier와 같은 파일명

JSON + ZIP 이미지
→ OCR 이미지-전사 텍스트 샘플
```

## 필라테스 프로그램

```text
labels.csv
├── image_path
└── pose_label

일반 이미지 파일
└── image_path가 가리키는 실제 이미지

CSV + 이미지 파일
→ 필라테스 동작 분류 샘플
```

필라테스 프로그램은 이미지 위치가 CSV에 직접 기록되어 있으므로 별도의 ZIP 인덱싱이나 ID 매핑이 필요하지 않습니다.

# 19. 주의할 점과 개선 사항

## 19.1 클래스당 첫 N개만 선택

현재 클래스 제한은 CSV에 등장한 순서대로 적용됩니다.

```python
if currentCount >= maxCount:
    continue
```

따라서 무작위 샘플링이 아닙니다.

보다 균형 있게 무작위로 선택하려면 CSV를 먼저 섞거나 클래스별 샘플링을 사용할 수 있습니다.

```python
df = (
    df.groupby("pose_label", group_keys=False)
      .apply(
          lambda group: group.sample(
              n=min(
                  len(group),
                  cfg["max_image_samples_per_class"]
              ),
              random_state=cfg["seed"]
          )
      )
      .reset_index(drop=True)
)
```

## 19.2 이미지 전체를 한 번에 메모리에 보관

다음 리스트에 모든 PIL 이미지가 들어갑니다.

```python
imageData = []
```

이미지가 많거나 해상도가 높으면 메모리 사용량이 크게 증가할 수 있습니다.

OCR 코드처럼 다음 방식을 고려할 수 있습니다.

* 청크 단위 처리

* 이미지 경로 기반 Dataset 구성

## 19.3 이미지 파일 하나의 오류가 전체 처리를 중단

`loadImageData()` 전체가 하나의 `try-except`로 감싸져 있습니다.

```python
try:
    for ...:
        image = Image.open(imagePath)
except Exception:
    return None
```

따라서 이미지 하나가 없거나 손상되면 전체 이미지 데이터 로드가 중단됩니다.

개별 이미지 단위 예외 처리가 더 안전합니다.

```python
for i in range(len(df)):
    try:
        with Image.open(imagePath) as img:
            image = img.convert("RGB").copy()
    except Exception as e:
        print(f"이미지 처리 오류: {imagePath} - {e}")
        continue
```

## 19.4 PIL 이미지 파일 핸들 관리

```python
Image.open(imagePath).convert("RGB")
```

이미지가 많으면 파일 핸들이 늦게 해제될 가능성이 있습니다.

```python
with Image.open(imagePath) as img:

```

# 20. 핵심 요약

이 프로그램의 핵심 목적은 다음과 같습니다.

> `labels.csv`의 `image_path`를 이용해 필라테스 이미지를 읽고, `pose_label`을 이미지 정답으로 사용하여 이미지-텍스트 학습 샘플을 생성합니다. 이후 Alpaca 형식의 텍스트 데이터와 동일한 7개 컬럼 구조로 통합하고 Hugging Face Hub에 업로드합니다.

데이터 연결 구조는 다음과 같습니다.

```text
labels.csv 한 행
├── image_path → 실제 이미지 파일
└── pose_label → 정답 동작명
              ↓
Hugging Face Dataset 한 행
├── image
├── instruction
├── input
├── output
├── source
├── label
└── modality
```

최종 Dataset은 다음 두 작업을 함께 지원합니다.

```text
필라테스 이미지 + 지시문
→ 필라테스 동작명과 설명

텍스트 instruction + input
→ 텍스트 output
```

다만 현재 설명 문장은 라벨을 넣은 고정 템플릿이므로, 실질적으로는 **필라테스 동작 이미지 분류 데이터와 일반 텍스트 지시 데이터를 하나의 멀티모달 Dataset으로 통합하는 코드**라고 보는 것이 가장 정확합니다.
