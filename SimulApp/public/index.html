📋 학습 시작 전 확인사항

i
DATASET_REPO — 기본 샘플 학습데이터가 설정되어 있습니다. 그대로 사용해도 됩니다. 동일한 포맷의 본인 HuggingFace 데이터셋이 있다면 교체할 수 있습니다.
!
MAX_TRAIN_SAMPLES — 기본값은 전체 데이터를 학습합니다. 처음 테스트 시 20 을 입력하면 20건만 빠르게 학습해서 동작을 확인할 수 있습니다.
✓
학습 결과는 서버 내 ./models 폴더에 자동 저장됩니다. 완료 후 시뮬레이션 결과 탭에서 손실 그래프와 학습 이력을 확인하세요.
✓
저장된 모델(어댑터)은 모델 병합 탭에서 [폴더 목록] 버튼으로 바로 선택할 수 있습니다.

📋 필수 데이터셋 포맷 — HuggingFace Dataset 컬럼 구조

컬럼명타입필수설명예시 값
modality string·classes ✅ 데이터 종류 구분 text_only  |  image_text
image Image 조건부 PIL 이미지 객체 (modality=image_text 일 때 필수, text_only는 None) PIL.Image / None
instruction string ✅ 모델에게 내릴 지시문 (system 역할 보조) 이미지를 보고 동작명을 분류하세요.
input string ✅ 사용자 입력 프롬프트 (질문 또는 보조 지시) 동작명과 설명을 한국어로 답하세요.
output string ✅ 모델이 생성해야 할 정답 (학습 label) bridge 설명: 이 이미지는 bridge 동작 예시입니다.
source string·classes 권장 데이터 출처 (추적·필터링 용도) pilates_dataset  |  generated  |  OCR_handwriting
label string·classes 권장 분류 레이블 (이미지 클래스명 등) bridge  |  plank  |  teaser

⚠ text_only 행은 image 컬럼이 None이어도 됩니다.  |  HuggingFace Hub 레포에 위 구조로 업로드된 데이터셋 ID를 DATASET_REPO에 입력하세요.

📋 GGUF 변환 전 체크리스트 — 아래 3가지는 반드시 입력하세요

1
GGUF_SOURCE_REPO — GGUF로 변환하고 싶은 본인의 HuggingFace 병합 모델 주소를 입력합니다. (모델 병합 탭의 MERGED_MODEL_REPO와 동일한 값)

GGUF_HF_REPO — 변환된 GGUF 파일을 업로드할 본인의 HuggingFace Hub 레포 주소를 입력합니다. (예: yourHFID/my-model-gguf)

GGUF_LOCAL_DIR선택 — 변경 불필요 비워두면 자동으로 경로가 생성됩니다.
