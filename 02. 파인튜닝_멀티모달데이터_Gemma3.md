# Gemma 3 멀티모달 LoRA 파인튜닝

`02. 파인튜닝_멀티모달데이터_Gemma3.py`는 Hugging Face Hub의 통합 멀티모달 데이터셋을 불러와 **Gemma 3 이미지-텍스트 모델을 LoRA 방식으로 지도 미세 조정(SFT)** 하는 스크립트입니다.

학습 실행 정보와 스텝별 손실은 MySQL에 기록하고, TensorBoard 로그·체크포인트·LoRA 어댑터·학습 손실 그래프를 실행별 출력 폴더에 저장합니다.

> [!IMPORTANT]
> 파일 설명에는 QLoRA라고 되어 있지만 현재 코드에서는 `quantization_config=quantConfig`가 주석 처리되어 있습니다. 따라서 실제 실행 방식은 **4비트 QLoRA가 아니라 BF16 LoRA**입니다.

![Gemma 3 파인튜닝 전체 흐름](images/gemma3_finetuning_pipeline.png)

---

## 1. 주요 기능

- Hugging Face 통합 데이터셋 로드
- `text_only`, `image_text` 샘플의 8:2 학습·평가 분할
- Gemma 3 대화 템플릿 적용
- 텍스트 전용, 이미지+텍스트, 혼합 배치 처리
- `input_ids`, `attention_mask`, `pixel_values`, `labels` 생성
- Gemma 3 전체 Linear layer에 LoRA 적용
- 에폭별 평가와 최적 `eval_loss` 체크포인트 선택
- 학습 스텝별 손실을 MySQL에 기록
- TensorBoard 로그와 `training_loss.png` 저장
- LoRA 어댑터와 Processor 저장

## 4. 입력 Dataset 구조

통합 Dataset은 다음 7개 컬럼을 가진다고 가정합니다.

| 컬럼 | 설명 |
|---|---|
| `modality` | `text_only` 또는 `image_text` |
| `image` | PIL 이미지 또는 `None` |
| `instruction` | 작업 지시문 |
| `input` | 사용자 입력 또는 출력 형식 안내 |
| `output` | 모델이 생성해야 할 정답 |
| `source` | 데이터 출처 |
| `label` | 이미지 분류 라벨 |

### 4.1 텍스트 전용 샘플

```python
{
    "modality": "text_only",
    "image": None,
    "instruction": "질문에 답하세요.",
    "input": "대한민국의 수도는?",
    "output": "서울입니다.",
    "source": "generated",
    "label": ""
}
```

### 4.2 이미지+텍스트 샘플

```python
{
    "modality": "image_text",
    "image": <PIL.Image.Image>,
    "instruction": "이미지를 보고 필라테스 동작명을 분류하세요.",
    "input": "동작명과 설명을 한국어로 답하세요.",
    "output": "동작명: Hundred\n설명: 이 이미지는 Hundred 동작 예시입니다.",
    "source": "pilates_dataset",
    "label": "Hundred"
}
```

모델 입력 변환에 직접 사용되는 필드는 `modality`, `image`, `instruction`, `input`, `output`입니다. `source`와 `label`은 현재 학습 Tensor에는 직접 포함되지 않습니다.

## 5. Dataset 행이 Tensor로 변환되는 과정

![Dataset에서 Tensor로 변환](images/dataset_to_tensors.png)

### 5.1 `buildMessages()`

Dataset 한 행을 Gemma 3 대화 형식으로 바꿉니다.

```python
[
    {
        "role": "system",
        "content": [
            {
                "type": "text",
                "text": "당신은 멀티모달 도우미입니다..."
            }
        ]
    },
    {
        "role": "user",
        "content": [
            {"type": "image", "image": image},
            {
                "type": "text",
                "text": "이미지를 보고 동작명을 분류하세요.\n\n[입력]\n한국어로 답하세요."
            }
        ]
    },
    {
        "role": "assistant",
        "content": [
            {
                "type": "text",
                "text": "동작명: Hundred"
            }
        ]
    }
]
```

텍스트 전용 샘플에서는 사용자 메시지의 이미지 항목이 제외됩니다.

### 5.2 Chat template 적용

```python
text = processor.apply_chat_template(
    messages,
    tokenize=False,
    add_generation_prompt=False
)
```

개념적으로 다음과 같은 Gemma 대화 문자열이 만들어집니다.

```text
<start_of_turn>system
당신은 멀티모달 도우미입니다.
<end_of_turn>

<start_of_turn>user
<image>
이미지를 보고 동작명을 분류하세요.
<end_of_turn>

<start_of_turn>model
동작명: Hundred
<end_of_turn>
```

`add_generation_prompt=False`를 사용하는 이유는 학습 메시지에 Assistant 정답이 이미 포함되어 있기 때문입니다. 이를 `True`로 설정하면 정답 뒤에 새로운 모델 응답 시작 표시가 다시 붙을 수 있습니다.

### 5.3 Processor 처리

Processor는 Tokenizer와 Image Processor를 함께 포함합니다.

```python
batch = processor(
    text=allTexts,
    images=nestedImages,
    return_tensors="pt",
    padding=True
)
```

텍스트는 다음 Tensor로 변환됩니다.

```text
input_ids
attention_mask
token_type_ids
```

이미지는 리사이즈·정규화된 뒤 다음 Tensor가 됩니다.

```text
pixel_values
```

## 6. 최종 모델 입력 Tensor

일반적인 배치 구조는 다음과 같습니다.

```python
{
    "input_ids":      Tensor[batch_size, sequence_length],
    "attention_mask": Tensor[batch_size, sequence_length],
    "token_type_ids": Tensor[batch_size, sequence_length],
    "pixel_values":   Tensor[number_of_images, channels, height, width],
    "labels":         Tensor[batch_size, sequence_length]
}
```

각 Tensor의 역할은 다음과 같습니다.

| Tensor | 역할 |
|---|---|
| `input_ids` | 텍스트와 특수 토큰을 정수 ID로 표현 |
| `attention_mask` | 실제 토큰은 1, 패딩은 0 |
| `token_type_ids` | 모델이 사용하는 토큰 유형 정보 |
| `pixel_values` | 정규화된 실제 이미지 픽셀 |
| `labels` | 다음 토큰 예측 손실 계산용 정답 |

`input_ids` 안에는 이미지 자체가 들어가지 않습니다. 이미지가 삽입될 위치를 나타내는 특수 토큰이 들어가며, 실제 이미지는 별도의 `pixel_values` Tensor로 전달됩니다.

## 7. 세 가지 배치 처리 사례

![세 가지 배치 처리 사례](images/batch_cases.png)

엄밀하게 현재 데이터셋에는 순수한 “이미지만 있는 샘플”은 없습니다. 이미지 샘플에도 instruction/input 텍스트가 붙기 때문에 세 경우는 다음과 같습니다.

### 7.1 텍스트 전용 배치

```text

이미지 수: 0
```

```python
input_ids.shape      == [2, L]
attention_mask.shape == [2, L]
labels.shape         == [2, L]
pixel_values         == 없음
```

Processor 호출:

```python
processor(
    text=allTexts,
    return_tensors="pt",
    padding=True
)
```

### 7.2 이미지+텍스트 전용 배치

```text
샘플 수: 2
이미지 수: 2
```

```python
input_ids.shape      == [2, L]
attention_mask.shape == [2, L]
labels.shape         == [2, L]
pixel_values.shape   == [2, C, H, W]
```

Processor 호출:

```python
processor(
    text=allTexts,
    images=[[image1], [image2]],
    return_tensors="pt",
    padding=True
)
```

### 7.3 혼합 배치

예를 들어 배치 순서가 다음과 같다고 가정합니다.

```text
0: image_text
1: text_only
2: image_text
```

코드는 이를 분리합니다.

```python
imgIdx = [0, 2]
txtIdx = [1]
```

이미지 서브배치와 텍스트 서브배치를 각각 Processor로 변환한 뒤, 시퀀스 길이를 동일하게 패딩하고 원래 순서로 복원합니다.

```python
input_ids.shape      == [3, Lmax]
attention_mask.shape == [3, Lmax]
labels.shape         == [3, Lmax]
pixel_values.shape   == [2, C, H, W]
```

모든 샘플에는 `input_ids`가 있지만, `pixel_values`는 실제 이미지가 있는 샘플 수만큼만 존재합니다.

## 8. `labels` 생성과 `-100` 마스킹

```python
labels = batch["input_ids"].clone()
```

Causal Language Model은 앞의 토큰을 이용해 다음 토큰을 예측하므로 입력 시퀀스를 복사해 정답 시퀀스로 사용합니다. 모델 내부에서는 logits와 labels가 한 토큰씩 이동되어 비교됩니다.

`.clone()`은 `labels`를 변경할 때 원본 `input_ids`까지 함께 변경되는 것을 방지합니다.

### 8.1 패딩 토큰 제외

```python
labels[labels == padTokenId] = -100
```

문장 길이를 맞추기 위해 추가한 패딩은 학습 정답이 아니므로 loss 계산에서 제외합니다.

### 8.2 이미지 토큰 제외

```python
labels[labels == imageTokenId] = -100
labels[labels == 262144] = -100
```

이미지 특수 토큰은 실제 텍스트 생성 정답이 아니므로 loss 계산에서 제외합니다.

PyTorch의 언어 모델 손실은 일반적으로 `-100`을 `ignore_index`로 처리합니다.

```text
일반 label → loss 계산
-100       → loss 계산 제외
```

> [!NOTE]
> 현재 코드는 패딩과 이미지 토큰만 제외합니다. 시스템 메시지와 사용자 질문 토큰도 loss 계산에 포함됩니다. Assistant 답변에만 loss를 적용하려면 response-only masking을 추가해야 합니다.

## 9. Gemma 3 Tokenizer

Tokenizer는 별도로 직접 로드하지 않고 `AutoProcessor` 내부에서 가져옵니다.

```python
processor = AutoProcessor.from_pretrained(
    cfg["base_model"]
)

tokenizer = processor.tokenizer
```

Gemma 3는 SentencePiece 계열의 Gemma Tokenizer를 사용하며, 일반 텍스트 subword와 다음 특수 토큰을 함께 처리합니다.

```text
BOS / EOS / PAD
대화 시작·종료 토큰
user / model 역할 토큰
이미지 위치 특수 토큰
```

확인 예시:

```python
print(type(processor.tokenizer))
print(processor.tokenizer.__class__.__name__)
print(processor.tokenizer.vocab_size)
print(processor.tokenizer.special_tokens_map)
```

문장 토큰화 결과 확인:

```python
text = "이미지를 보고 필라테스 동작명을 분류하세요."

print(processor.tokenizer.tokenize(text))
print(
    processor.tokenizer.encode(
        text,
        add_special_tokens=False
    )
)
```

## 10. 모델 내부 멀티모달 결합

텍스트와 이미지는 서로 다른 경로로 처리됩니다.

```text
input_ids
    ↓
Text Embedding
    ↓
텍스트 임베딩
```

```text
pixel_values
    ↓
Vision Encoder
    ↓
이미지 패치 특징
    ↓
Multimodal Projector
    ↓
언어 모델 hidden size의 이미지 임베딩
```

이미지 특수 토큰이 있는 위치에 Projector가 만든 이미지 임베딩이 연결됩니다.

```text
텍스트 임베딩
+
이미지 임베딩
    ↓
Gemma Transformer
    ↓
logits
```

`logits`의 개념적 형태는 다음과 같습니다.

```python
logits.shape == [
    batch_size,
    sequence_length,
    vocabulary_size
]
```

각 위치에서 다음 토큰이 될 수 있는 vocabulary 전체의 점수를 출력합니다.

## 11. Loss와 LoRA 학습

모델은 logits와 labels를 한 칸 이동하여 Causal LM loss를 계산합니다.

```text
현재까지 입력한 토큰 → 다음 토큰 예측
```

```text
"동작명"           → ":"
"동작명:"          → "Hundred"
"동작명: Hundred"  → 다음 토큰
```

손실이 계산되면 역전파가 수행됩니다.

```text
loss
    ↓
backward
    ↓
gradient accumulation
    ↓
optimizer step
    ↓
LoRA 어댑터 파라미터 업데이트
```

LoRA에서는 베이스 모델의 기존 가중치는 일반적으로 고정하고, 각 Linear layer에 추가된 저랭크 행렬만 학습합니다.

```text
기존 출력: W x
LoRA 보정: B A x × scaling
최종 출력: W x + B A x × scaling
```

현재 설정:

```python
LoraConfig(
    r=16,
    lora_alpha=32,
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",
    target_modules="all-linear"
)
```

`target_modules="all-linear"`은 모델의 모든 Linear layer를 대상으로 하므로 언어 모델, 멀티모달 Projector, Vision Encoder 내부 Linear layer까지 포함될 가능성이 있습니다.

## 12. QLoRA 설정 상태

코드에는 4비트 양자화 구성이 정의되어 있습니다.

```python
quantConfig = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torchDtype,
    bnb_4bit_use_double_quant=False
)
```

하지만 모델 로드에서 다음 줄이 주석 처리되어 있습니다.

```python
# quantization_config=quantConfig,
```

따라서 현재 실행은 다음 방식입니다.

```text
베이스 모델: BF16
파인튜닝: LoRA
```

실제 QLoRA로 실행하려면 모델 로드 시 `quantization_config=quantConfig`를 전달해야 합니다.

## 13. 학습 설정

주요 `SFTConfig` 설정은 다음과 같습니다.

| 설정 | 의미 |
|---|---|
| `num_train_epochs` | 전체 학습 에폭 |
| `per_device_train_batch_size` | GPU당 배치 크기 |
| `gradient_accumulation_steps` | Gradient 누적 횟수 |
| `gradient_checkpointing=True` | 메모리 절약 |
| `max_seq_length` | 최대 텍스트 시퀀스 길이 |
| `packing=False` | 여러 샘플을 한 시퀀스로 묶지 않음 |
| `eval_strategy="epoch"` | 에폭마다 평가 |
| `save_strategy="epoch"` | 에폭마다 체크포인트 저장 |
| `load_best_model_at_end=True` | 최저 `eval_loss` 모델 복원 |
| `remove_unused_columns=False` | 이미지 등 원본 컬럼 유지 |
| `skip_prepare_dataset=True` | Custom Collator가 직접 전처리 |

실효 배치 크기는 대략 다음과 같습니다.

```text
실효 배치 크기
= GPU당 배치 크기
× gradient accumulation
× GPU 수
```

```python
split_ds = dataset.train_test_split(
```

## 18. 주요 주의사항과 개선점

### 18.1 현재는 QLoRA가 아님

`quantization_config`가 주석 처리되어 있으므로 BF16 LoRA입니다.

### 18.2 CUDA가 없으면 오류 가능

다음 호출 전에 CUDA 사용 가능 여부를 확인해야 합니다.

```python
torch.cuda.get_device_capability()
```

### 18.3 BF16 지원 검사 필요

현재 `bf16=torch.cuda.is_available()`는 CUDA GPU가 BF16을 지원한다고 가정합니다. GPU Compute Capability를 기준으로 BF16과 FP16을 구분하는 편이 안전합니다.

### 18.4 DB의 `train_loss` 의미 불일치

현재 `train_loss` 컬럼에 `bestEvalLoss`를 저장합니다. 실제 학습 손실과 최적 평가 손실을 별도 컬럼으로 관리해야 합니다.

### 18.5 `SAVE_STEPS`가 실제 저장 주기에 적용되지 않음

현재 `save_strategy="epoch"`이므로 `SAVE_STEPS` 환경 변수는 실제 저장 주기에 영향을 주지 않습니다.

### 18.6 전체 메시지에 loss가 적용됨

시스템 메시지와 사용자 질문까지 학습 대상입니다. Assistant 답변만 학습하려면 response-only masking이 필요합니다.

### 18.7 혼합 배치의 추가 Processor 키 점검

현재 `_mergeMixedBatch()`는 다음 키를 중심으로 병합합니다.

```text
input_ids
attention_mask
token_type_ids
pixel_values
```

Transformers 또는 모델 버전에 따라 `image_sizes`, `image_grid_thw` 등의 추가 키가 필요할 수 있습니다.

### 18.8 평가 지표가 `eval_loss`에 한정됨

실제 필라테스 동작 분류 성능을 확인하려면 Accuracy, Precision, Recall, F1-score 또는 생성된 동작명과 `label`의 정확 일치율을 별도로 계산해야 합니다.

## 19. 핵심 요약

이 스크립트는 통합 멀티모달 Dataset의 각 행을 다음과 같이 처리합니다.

```text
Dataset 행
    ↓
system / user / assistant 메시지
    ↓
Gemma 3 Chat template
    ↓
텍스트 Tokenizer + 이미지 Processor
    ↓
input_ids + attention_mask + pixel_values + labels
    ↓
Gemma 3 Vision Encoder + Language Model
    ↓
Causal LM loss
    ↓
LoRA 어댑터 업데이트
```

텍스트는 `input_ids`, 이미지는 `pixel_values`로 별도 전달됩니다. 모델 내부에서 이미지 특징이 언어 모델 hidden size로 투영되고, 이미지 특수 토큰 위치에 결합되어 하나의 멀티모달 시퀀스로 처리됩니다.

현재 구현의 정확한 학습 방식은 다음과 같습니다.

> **Gemma 3 4B 멀티모달 모델을 BF16으로 로드하고, 텍스트 전용과 이미지+텍스트 데이터가 섞인 Dataset을 Custom Collator로 Tensor화하여 모든 Linear layer에 LoRA를 적용하는 지도 미세 조정 코드입니다.**
