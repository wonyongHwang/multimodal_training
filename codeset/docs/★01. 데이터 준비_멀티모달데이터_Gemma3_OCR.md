# 모델정의서: ★01. 데이터 준비_멀티모달데이터_Gemma3_OCR

## 개요
손글씨 OCR 이미지 데이터와 Alpaca 형식 텍스트 데이터를 통합하여 Hugging Face Hub에 업로드하는 데이터 전처리 노트북.

---

## 환경 변수 (.env)

| 변수명 | 설명 | 기본값 |
|---|---|---|
| `HF_TOKEN` | Hugging Face 인증 토큰 | - |
| `HF_DATASET_REPO` | 업로드할 HF 데이터셋 저장소 | `hyokwan/ocr_dataset` |
| `OCR_ROOT` | OCR 데이터셋 루트 경로 (ZIP·라벨 공통 기준) | `./dataset/05. 멀티모달 OCR/OCR손글씨` |
| `TEXT_JSON_PATH` | Alpaca 형식 텍스트 JSON 경로 | `./dataset/04. 멀티모달 샘플/text_samples/sample_data.json` |
| `MAX_OCR_SAMPLES` | OCR 샘플 최대 개수 (빈값=전체) | 빈값(전체) |
| `CHUNK_SIZE` | 청크 처리 단위 (JSON 파일 수) | `500` |
| `DATASET_DIR` | Arrow shard 저장/로드 경로 (pickle·Arrow 통합) | `./dataset/ocr_dataset` |
| `SEED` | 난수 시드 | `42` |
| `OCR_INSTR_TEXT` | OCR instruction 프롬프트 | `이미지에서 손글씨로 작성된 텍스트를 모두 인식하여 전사하세요.` |
| `OCR_INPUT_TEXT` | OCR input 프롬프트 | `이미지에 포함된 모든 텍스트를 공백으로 구분하여 순서대로 나열해주세요.` |

---

## 데이터 흐름

```
OCR JSON 라벨 + ZIP 이미지
        ↓ buildZipImageMap / collectJsonFiles
    청크 처리 → Arrow shard 직접 저장
        ↓ DATASET_DIR/shard_0000 ~ shard_NNNN
    load_from_disk + concatenate_datasets
        ↓
Alpaca 텍스트 JSON → text_dataset
        ↓
  concatenate_datasets → shuffle
        ↓
  push_to_hub (HF_DATASET_REPO)
```

---

## 주요 함수

| 함수명 | 설명 |
|---|---|
| `buildZipImageMap(zipRootDir)` | ZIP 파일을 재귀 탐색하여 이미지ID → (zip경로, 내부경로) 매핑 생성 |
| `collectJsonFiles(labelRoot)` | 라벨 루트에서 모든 JSON 파일 경로 수집 |
| `loadOcrImage(imgId, zipMap)` | ZIP 매핑으로 이미지 ID에 해당하는 PIL 이미지 로드 |

---

## 출력 데이터셋 스키마

| 컬럼 | 타입 | 설명 |
|---|---|---|
| `modality` | string | `image_text` 또는 `text_only` |
| `image` | Image | PIL 이미지 (텍스트 샘플은 None) |
| `instruction` | string | 모델 지시문 |
| `input` | string | 모델 입력 컨텍스트 |
| `output` | string | 정답 텍스트 |
| `source` | string | `OCR_handwriting` 또는 `generated` |
| `label` | string | `P.Paper`, `T.Tablet`, 또는 빈값 |
