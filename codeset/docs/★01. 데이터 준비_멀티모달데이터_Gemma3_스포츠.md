## .env 설정값

| `.env` 키 | 기본값 | 설명 |
|-----------|--------|------|
| `HF_TOKEN` | — | HuggingFace 액세스 토큰 |
| `HF_DATASET_REPO` | `hyokwan/multi_modal_sample` | 업로드할 HF 레포 |
| `IMAGE_ROOT` | `./dataset/04. 멀티모달 샘플/image_samples` | 이미지 파일 루트 경로 |
| `IMAGE_META_ROOT` | `./dataset/04. 멀티모달 샘플/image_samples/labels.csv` | 이미지 메타 CSV 경로 |
| `TEXT_JSON_PATH` | `./dataset/04. 멀티모달 샘플/text_samples/sample_data.json` | 텍스트 JSON 경로 |
| `MAX_IMAGE_SAMPLES_PER_CLASS` | (비어있으면 제한 없음) | 클래스당 최대 이미지 수 |
| `IMAGE_INSTRUCTION` | `이미지를 보고 필라테스 동작명을 분류하고 한 줄 설명을 제공하세요.` | 이미지 샘플 instruction |
| `IMAGE_INPUT` | `동작명과 간단한 설명을 한국어로 답하세요.` | 이미지 샘플 input |
| `SEED` | `42` | 셔플 시드 |

> `.env` 값만 바꾸면 이미지 경로·프롬프트·샘플 수 제한을 코드 수정 없이 조정 가능

## 데이터 흐름

```
TEXT_JSON_PATH (JSON)
    └─ pd.read_json → alpacaDf → textDataset

IMAGE_META_ROOT (CSV)
    └─ pd.read_csv → df
        └─ 이미지 루프 (MAX_IMAGE_SAMPLES_PER_CLASS 제한) → imageData → imageDataset

imageDataset + textDataset
    └─ cast(features) → concatenate → shuffle → push_to_hub(HF_DATASET_REPO)
```

## 알려진 이슈

| 이슈 | 원인 | 해결 |
|------|------|------|
| `concatenate_datasets` 타입 불일치 | pandas → `large_string` vs list → `string` 타입 충돌 (datasets 버전 업) | `textDataset.cast(imageDataset.features)` 선적용 |

## .py 스크립트 구조 (`★01. 데이터 준비_멀티모달데이터_Gemma3_스포츠.py`)

| 함수 | 역할 |
|------|------|
| `loadConfig()` | `.env` 전체 파싱 → 설정 딕셔너리 반환 |
| `loadTextData(cfg)` | 텍스트 JSON → Alpaca 포맷 `Dataset` 반환 |

| `mergeAndUpload(textDataset, imageDataset, cfg)` | 병합·캐스팅·셔플·Hub 업로드 |
| `main()` | 전체 파이프라인 실행 |

**실행:** `python "★01. 데이터 준비_멀티모달데이터_Gemma3_스포츠.py"`
