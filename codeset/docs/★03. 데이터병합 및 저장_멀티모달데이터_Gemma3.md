# 모델 정의서: ★03. 데이터병합 및 저장_멀티모달데이터_Gemma3.py

## 개요

| 항목 | 내용 |
|------|------|
| 파일명 | `codeset/★03. 데이터병합 및 저장_멀티모달데이터_Gemma3.py` |
| 목적 | LoRA 어댑터를 베이스 모델에 병합 → 로컬 저장 → HuggingFace Hub 업로드 → 추론 검증 |
| 실행 방법 | `python "★03. 데이터병합 및 저장_멀티모달데이터_Gemma3.py"` 또는 SimulApp 🔀 병합 탭 |

---

## 함수 구조

| 함수 | 역할 |
|------|------|
| `loadConfig()` | 스크립트 디렉토리 기준 `.env` 로드 → 설정 딕셔너리 반환 |
| `hfLogin(hfToken)` | HF 로그인 (실패 시 경고만 출력, 크래시 없음) |
| `loadBaseModel(cfg)` | 양자화 없이 fp16/bf16으로 베이스 모델 + 프로세서 로드 |
| `mergeAdapter(baseModel, adapterPath)` | PeftModel 로드 → `merge_and_unload()` → 병합 모델 반환 |
| `saveMerged(mergedModel, processor, mergedLocalDir)` | 양자화 잔재 검증 → `tie_weights()` → 로컬 저장 |
| `uploadToHub(mergedLocalDir, mergedModelRepo, hfToken)` | `HfApi.upload_folder()` → Hub 업로드 → 로컬 삭제 |
| `loadMergedFromHub(mergedModelRepo, hfToken)` | Hub에서 검증용 재로드 |
| `inferFromHub(question, inferModel, inferProcessor, ...)` | 텍스트/이미지+텍스트 추론 |
| `main()` | 전체 파이프라인 실행 |

---

## .env 설정값

| `.env` 키 | 기본값 | 설명 |
|-----------|--------|------|
| `HF_TOKEN` | `YOUR_HF_TOKEN` | HuggingFace 액세스 토큰 |
| `BASE_MODEL` | `google/gemma-3-4b-it` | 병합 대상 베이스 모델 ID |
| `ADAPTER_PATH` | `./models/gemma3_multimodal_lora_output` | 파인튜닝된 LoRA 어댑터 경로 |
| `MERGED_LOCAL_DIR` | `./models/gemma3_multimodal_merged` | 병합 모델 로컬 저장 경로 |
| `MERGED_MODEL_REPO` | `YOUR_HF_ID/multi_modal_model_g3` | HF Hub 업로드 레포 |
| `TEST_IMAGE_PATH` | (빈값) | 추론 테스트 이미지 경로 (선택) |

---

## 처리 흐름

```
loadBaseModel()
    └─ PeftModel.from_pretrained(ADAPTER_PATH)
        └─ merge_and_unload()
            ├─ saveMerged(MERGED_LOCAL_DIR)
            └─ uploadToHub(MERGED_MODEL_REPO)
                └─ loadMergedFromHub() → inferFromHub() (검증)
```
