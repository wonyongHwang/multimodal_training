# 모델 정의서: ★02. 파인튜닝_멀티모달데이터_Gemma3.py

## 개요

| 항목 | 내용 |
|------|------|
| 파일명 | `codeset/★02. 파인튜닝_멀티모달데이터_Gemma3.py` |
| 목적 | Gemma 3 4B-IT QLoRA 파인튜닝 (텍스트+이미지 멀티모달) |
| 출력 | LoRA 어댑터 (`OUTPUT_BASE_DIR/gemma3_multimodal_lora_<날짜시간>/`) |
| DB 연동 | 시뮬레이션 실행 이력(`simulation_runs`) + 스텝 손실 로그(`simulation_logs`) |

---

## 실행 방법

```bash
python "★02. 파인튜닝_멀티모달데이터_Gemma3.py"
```

하이퍼파라미터는 `codeset/.env` 또는 SimulApp 웹 화면에서 변경 가능 (코드 수정 불필요).

---

## 함수 구조

| 함수 | 역할 |
|------|------|
| `setKoreanFont()` | matplotlib 한글 폰트 설정 (맑은 고딕 우선) |
| `loadConfig()` | 스크립트 디렉토리의 `.env` 로드 → 전체 설정 딕셔너리 반환 |
| `getDbConn(dbCfg)` | MySQL 연결 객체 반환 |
| `getNextSimSeq(dbCfg)` | `MAX(sim_seq)+1` 조회로 다음 시뮬레이션 번호 발급 |
| `insertRunStart(simSeq, cfg)` | 학습 시작 시 `simulation_runs` INSERT (status=running) |
| `updateRunEnd(simSeq, trainResult, dbCfg, status, errorMsg)` | 학습 완료/실패 시 결과 UPDATE |
| `DbLogCallback` | `TrainerCallback` 서브클래스 — `on_log`로 스텝마다 `simulation_logs` INSERT |
| `buildMessages(example)` | 샘플 → 모달리티별 메시지 구조 변환 |
| `getCollateFn(processor, model)` | 전역 의존 없는 collate_fn 클로저 반환 |
| `plotTrainingLoss(trainer, outputDir)` | 손실 추이 PNG 저장 (`training_loss.png`) |
| `main()` | 전체 학습 파이프라인 실행 |

## .env 설정값

| `.env` 키 | 기본값 | 설명 |
|-----------|--------|------|
| `HF_TOKEN` | — | HuggingFace 액세스 토큰 |
| `DATASET_REPO` | `hyokwan/multi_modal_sample` | 학습에 사용할 HF 데이터셋 |
| `BASE_MODEL` | `google/gemma-3-4b-it` | 베이스 모델 ID |
| `OUTPUT_BASE_DIR` | `./models` | 어댑터 저장 루트 경로 |
| `MAX_TRAIN_SAMPLES` | (비어있으면 전체) | 학습 데이터 수 제한 |
| `MAX_EVAL_SAMPLES` | (비어있으면 전체) | 평가 데이터 수 제한 |
| `MAX_SEQ_LENGTH` | `768` | 최대 시퀀스 길이 |
| `PER_DEVICE_TRAIN_BATCH_SIZE` | `1` | GPU당 학습 배치 크기 |
| `PER_DEVICE_EVAL_BATCH_SIZE` | `1` | GPU당 평가 배치 크기 |
| `GRAD_ACCUM` | `4` | 그래디언트 누적 스텝 수 |
| `NUM_EPOCHS` | `5` | 학습 에폭 수 |
| `LEARNING_RATE` | `2e-4` | 학습률 |
| `LOGGING_STEPS` | `10` | 손실 로깅 간격 (스텝) |
| `SAVE_STEPS` | `100` | 체크포인트 저장 간격 (스텝) |
| `LORA_R` | `16` | LoRA rank |
| `LORA_ALPHA` | `32` | LoRA alpha |
| `LORA_DROPOUT` | `0.05` | LoRA dropout |
| `DB_HOST` | — | MySQL 호스트 |
| `DB_PORT` | `3306` | MySQL 포트 |
| `DB_NAME` | — | 데이터베이스명 |
| `DB_USER` | — | DB 사용자명 |
| `DB_PASSWORD` | — | DB 비밀번호 |

## DB 스키마

### simulation_runs

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | INT AUTO_INCREMENT | PK |
| `sim_seq` | INT UNIQUE | 시뮬레이션 순번 |
| `status` | VARCHAR(20) | `running` / `completed` / `failed` |
| `run_timestamp` | DATETIME | 학습 시작 시각 |
| `end_timestamp` | DATETIME | 학습 종료 시각 |
| `model_name` | VARCHAR(255) | 베이스 모델 ID |
| `dataset_repo` | VARCHAR(255) | 데이터셋 레포 |
| `output_dir` | VARCHAR(512) | 어댑터 저장 경로 |
| `tensorboard_dir` | VARCHAR(512) | TensorBoard 로그 경로 |
| `num_epochs` | INT | 에폭 수 |
| `learning_rate` | FLOAT | 학습률 |
| `lora_r` | INT | LoRA rank |
| `lora_alpha` | INT | LoRA alpha |
| `lora_dropout` | FLOAT | LoRA dropout |
| `max_seq_length` | INT | 최대 시퀀스 길이 |
| `grad_accum` | INT | 그래디언트 누적 |
| `batch_size` | INT | 학습 배치 크기 |
| `logging_steps` | INT | 로깅 간격 |
| `save_steps` | INT | 저장 간격 |
| `train_loss` | FLOAT | 최종 학습 손실 |
| `runtime_sec` | FLOAT | 총 소요 시간(초) |
| `samples_per_sec` | FLOAT | 초당 처리 샘플 수 |
| `global_step` | INT | 총 학습 스텝 수 |
| `error_message` | TEXT | 오류 내용 (실패 시) |

### simulation_logs

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | INT AUTO_INCREMENT | PK |
| `sim_seq` | INT | 시뮬레이션 번호 |
| `step` | INT | 학습 스텝 |
| `loss` | FLOAT | 스텝 손실 |
| `learning_rate` | FLOAT | 스텝 학습률 |
| `epoch` | FLOAT | 현재 에폭 |
| `log_timestamp` | DATETIME | 로그 기록 시각 |

## 출력 경로 구조

```
OUTPUT_BASE_DIR/
└── gemma3_multimodal_lora_<YYYYMMDD_HHMMSS>/
    ├── adapter_config.json
    ├── adapter_model.safetensors
    ├── training_loss.png         (손실 추이 그래프)
    └── tensorboard/              (TensorBoard 로그)
```

TensorBoard 확인:
```bash
tensorboard --logdir <tensorboard_dir>
```
