## 주요 동작 — getCollateFn / collateFn

### 배치 처리 3-way 분기

`collateFn` 은 배치 내 샘플 구성에 따라 3가지 경로로 분기합니다.

| 경우 | `nestedImages` 구조 | 처리 방식 |
|------|-------------------|----------|
| 전부 text-only | `[None, None, ...]` | `processor(text=...)` — images 없이 호출 |
| 전부 image-text | `[[img1], [img2], ...]` | `processor(text=..., images=nestedImages)` 단일 호출 |
| 혼합 | `[[img1], None, [img3], ...]` | 분리 처리 후 `_mergeMixedBatch` 로 병합 |

### 핵심: `[[img]]` 중첩 리스트 구조

`make_nested_list_of_images` 의 동작:
- `[img1, img2, ..., imgN]` (flat) → 이미지 N개짜리 **배치 1건** → `len = 1 ≠ N` → 에러
- `[[img1], [img2], ..., [imgN]]` (중첩) → 1이미지씩 **N개 배치** → `len = N == N` → 정상

### _mergeMixedBatch 동작

혼합 배치 처리 시 호출되는 내부 함수:
1. `imgBatch = processor(text=imgTexts, images=imgImages)` — 이미지 샘플 서브배치
2. `txtBatch = processor(text=txtTexts)` — 텍스트 샘플 서브배치
3. 두 배치의 `input_ids`, `attention_mask`, `token_type_ids` 를 maxSeqLen 으로 패딩 후 원본 인덱스 순서로 합산
4. `pixel_values` 는 imgBatch 에서 그대로 가져옴 (이미지 토큰 ID 로 위치 매핑됨)
